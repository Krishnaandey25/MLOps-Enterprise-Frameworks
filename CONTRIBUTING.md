# Contributing to the AI Projects

Thank you for your interest in contributing to our open-source library. We follow strict 'CTO Standard' engineering practices.

## Coding Standards
- Use **Type Hinting** for all function arguments and return values.
- Adhere to **PEP 8** style guidelines.
- Use **Abstract Base Classes (ABC)** and **Factory Patterns** for extensibility.
- Document all classes and methods using **Google-style docstrings**.

## Pull Request Process
1. Create a feature branch.
2. Ensure all tests pass via `make test`.
3. Format code via `make format`.
4. Submit PR with a clear description of changes.

## Development Environment
- Install dependencies: `make install`
- Run linting: `make lint`
